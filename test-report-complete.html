REG PRUEBAS DE REGRESIÓN — BUGS CONFIRMADOS RESUELTOS 5 regresiones verificadas · Never-regress lista
🐛
CRITICAL   BUG-01: Workspace colapsa al escribir en textarea — RESUELTO
Root cause: .agent-card tiene onclick="toggleAgent(id)". Click en textarea burbujea al card, toggleAgent cierra el workspace. Los 15 agentes afectados. Fix: onclick="event.stopPropagation()" en .agent-workspace cubre todos los inputs, textareas, labels y selects de cada agente sin tocar individualmente los 8 renderers custom.
Commit: 73e3e53 · marketing.html línea ~944 · Verificado: MTX-REG-001 PASS · MTX-AGT-003 PASS
📍
CRITICAL   BUG-02: MetaFollow se superponía con ícono de Chat — RESUELTO
Root cause: Ambos botones position:fixed en la misma esquina sin suficiente separación. Chat: bottom:108px (44px tall). MetaFollow (80px tall): necesitaba bottom ≤ 28px para no solapar. Estaba en 16px → subido a 20px da gap visual de 8px. Además: browsers tenían cached la versión vieja → fix de cache-bust ?v=3→v=4 necesario.
Commit: 73e3e53 + 589ead0 · salescoach.js bottom:20px · Verificado: MTX-REG-002 PASS · MTX-MFW-002 PASS
🗄️
HIGH   BUG-03: Cache stale de JS — usuarios veían versiones antiguas — RESUELTO
Browsers cacheaban salescoach.js y teamchat.js sin versión. Los fixes de MetaFollow y DMs no eran visibles hasta hard-refresh manual. Solución: ?v=3 y luego ?v=4 en todos los script tags. Aplicado en 11 páginas HTML. También app.js y agent.js versionados por primera vez.
Commit: 589ead0 + bfbba3a · 11 archivos HTML · Verificado: MTX-REG-003 PASS · MTX-INF-007 PASS
💬
CRITICAL   BUG-04: DM storage key no canónica — conversaciones duplicadas — RESUELTO
Si User A iniciaba DM a B y User B respondía, cada uno usaba una key diferente (con su propio ID primero), creando 2 conversaciones separadas. Fix: sort() ambos IDs antes de construir la key → siempre mtx_team_chat_dm_{menor}_{mayor}. Verificado: dmStorageKey('zzz','aaa') === dmStorageKey('aaa','zzz').
Commit: cbe40c1 · teamchat.js · dmStorageKey() · Verificado: MTX-REG-004 PASS · MTX-CHT-006 PASS
🔄
HIGH   BUG-05: Cross-tab DM sync contaminaba a otros usuarios — RESUELTO
El evento storage en teamchat.js sincronizaba todos los mensajes DM a todas las pestañas sin filtrar por destinatario. User Z podía ver DMs entre X e Y. Fix: campo is_dm:true + to_user en mensajes DM, filtrado en el listener de storage para solo procesar DMs donde to_user === currentUser.id.
Commit: cbe40c1 · teamchat.js · sendMessage() + storage listener · Verificado: MTX-REG-005 PASS · MTX-CHT-008
